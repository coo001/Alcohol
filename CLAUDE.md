# Project Overview

이 프로젝트는 **사용자 시점(POV)으로 술자리에 앉아 직접 술을 마시는 듯한 느낌을 주는 1인칭 인터랙션 웹앱**이다.

핵심은 채팅이 아니라 **몰입감 있는 술자리 체험**이다.  
사용자는 자신의 시점에서 테이블, 술잔, 술병, 안주, 주변 분위기를 보고 직접 술을 따르고, 잔을 들고, 마시고, 건배하는 흐름을 경험해야 한다.

지원 음료는 다음을 포함한다.

- 소주
- 맥주
- 양주
- 위스키
- 음료

이 앱은 실제 음주를 권장하는 서비스가 아니라, **술자리의 분위기와 감각을 디지털 인터랙션으로 재현하는 장난감형 체험 서비스**다.

---

# Product Goal

사용자가 앱을 켰을 때 바로 이런 느낌을 받아야 한다.

- 내가 술자리에 앉아 있다
- 내 앞에 잔과 병이 있다
- 내가 직접 따른다
- 내가 직접 마신다
- 주변이 실제 술자리처럼 반응한다

즉, 목표는 정보 제공이 아니라 **감각적 체험**이다.

---

# Core Experience

반드시 다음 UX를 중심으로 구현한다.

1. **1인칭 시점**
   - 사용자의 얼굴이나 전신은 보여주지 않는다
   - 사용자 시점에서 테이블 위만 보이게 한다
   - 필요하면 손 정도만 보이게 한다

2. **직접 마시는 인터랙션**
   - 병을 따른다
   - 잔을 든다
   - 마신다
   - 건배한다
   - 다시 따른다

3. **음료별 차이**
   - 소주, 맥주, 위스키, 양주, 음료마다
   - 잔 모양
   - 병 형태
   - 따르는 느낌
   - 마시는 텍스트/연출
   - 분위기
   가 달라야 한다

4. **분위기 몰입**
   - 포차
   - 호프집
   - 바
   - 집술
   같은 배경 분위기를 지원한다

5. **점진적 변화**
   - 여러 번 마실수록 화면, 텍스트, 분위기가 미세하게 변한다
   - 단, 과장된 취함 표현보다는 분위기 변화 중심으로 간다

# Non-Goals

초기 버전에서는 아래를 하지 않는다.

- 복잡한 로그인/회원가입
- 결제
- 실제 주류 구매 기능
- 소셜 피드
- 멀티플레이
- 복잡한 3D 엔진
- 과도한 백엔드 구조
- LLM 중심 채팅앱 구조

핵심은 **빠르게 몰입형 데모를 만드는 것**이다.

---

# Design Direction

디자인은 예쁘기만 한 UI보다 **분위기와 몰입**이 중요하다.

지향점:
- 어두운 배경
- 조용한 조명 또는 술집 무드
- 과하지 않은 네온/앰비언트 톤
- 모바일 우선
- 버튼은 직관적
- 시선은 항상 잔과 병 중심

피해야 할 것:
- 지나치게 게임 UI처럼 보이는 것
- 화려한 대시보드 스타일
- 밝고 업무용 같은 느낌
- 정보가 너무 많은 화면

---

# Technical Direction

기술 우선순위는 다음과 같다.

1. **빠른 프로토타이핑**
2. **단순한 상태 관리**
3. **부드러운 애니메이션**
4. **확장 가능한 데이터 구조**

권장 스택:
- Next.js
- React
- Tailwind
- Framer Motion

초기 구현 원칙:
- mock 데이터 우선
- 이미지 없어도 CSS shape와 layout으로 먼저 구현
- 파일 수 최소화
- 컴포넌트는 과분리하지 않는다
- 기능이 확인된 뒤 구조 정리

# Interaction Rules

구현 시 반드시 지킬 것:

- 사용자는 항상 “내가 마신다”는 감각을 받아야 한다
- 액션 하나마다 화면 반응이 있어야 한다
- 클릭했는데 아무 일도 안 일어나면 안 된다
- 상태 변화는 명확해야 한다
- 한 번 마실 때마다 잔의 상태가 바뀌어야 한다
- 따르기 전/후, 마시기 전/후 상태를 분리한다
- 애니메이션은 짧고 부드럽게 한다
- 재미보다도 먼저 “손맛”이 느껴지게 한다

---

# Tone

이 앱의 톤은 아래에 가깝다.

- 감각적
- 은근히 재밌음
- 몰입감 있음
- 가볍지만 싸구려 같지 않음
- 술자리 분위기를 재현하지만 과장된 만취 개그는 지양

문구 스타일:
- 짧고 직관적
- 너무 설명적이지 않음
- 버튼 문구도 분위기를 해치지 않게 구성

---

# Random Event Direction

랜덤 이벤트는 작고 자연스러워야 한다.

예시:
- 누군가 잔을 채워줌
- “짠” 하자는 제안
- 얼음 추가
- 안주가 새로 나옴
- 주변 소리가 잠깐 커짐
- 한마디 툭 던지는 반응

이벤트는 UX를 방해하지 말고 몰입을 보조해야 한다.

# Implementation Priorities

항상 아래 순서로 작업한다.

1. 기본 화면 구성
2. 잔/병/테이블 상태 표현
3. 따르기/들기/마시기 인터랙션
4. 음료별 차이 반영
5. 분위기별 차이 반영
6. 반응 텍스트와 랜덤 이벤트
7. 디테일 polish

---

# Definition of Done

첫 버전 완료 기준:

- 사용자가 음료를 선택할 수 있다
- 사용자가 분위기를 선택할 수 있다
- 메인 화면에서 1인칭 술자리처럼 보인다
- 술을 따를 수 있다
- 잔을 들 수 있다
- 마실 수 있다
- 건배할 수 있다
- 음료별 연출 차이가 있다
- 여러 번 상호작용할수록 분위기가 조금씩 바뀐다
- 전체적으로 “술 마시는 느낌”이 난다

---

# Important Constraint

구현 중 판단이 필요하면 항상 아래 기준을 따른다.

**채팅 기능보다 POV 체험을 우선한다.**  
**복잡한 기능보다 몰입감 있는 작은 인터랙션을 우선한다.**  
**예쁜 구조보다 실제로 재밌는 손맛을 우선한다.**
